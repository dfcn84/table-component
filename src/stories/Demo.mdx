# Table Component Demo
This is the documentation for the `Table` component.

## Table Data Object Description
The `tableData` object represents a structured dataset with no
  fixed attributes to allow flexibility

```
{
  tableName: "sub-data",
  selectType: "checkbox",
  headers: {
    brn: { title: "BRN" },
    companyName: { title: "Company Name" }
  },
  data: [
    {
      brn: "198702333K",
      companyName: "Blue Ocean International"
    },
    {
      brn: "198702333K",
      companyName: "Red Electronics"
    },
    {
      brn: "196700335H",
      companyName: "Yellow Gaming"
    },
    {
      brn: "196800306E",
      companyName: "Purple Automobiles"
    },
    {
      brn: "199131124V",
      companyName: "Diamond Interiors Holdings"
    },
    {
      brn: "200201624D",
      companyName: "Omnichannel Solutions International"
    }
  ]
}
```

### Properties
- `tableName`: String

  Description: Specifies the name or identifier of the table.
- `collapseCol`: Boolean
  
  Description: If the data row column need to be collapsed on mobile
      view.
- `mobileHeader`: String
    
  Description: Table header for mobile view
- `selectType`: Enum `"checkbox" | "radio"`

  Description: Set this value if selection is needed.
- `headers`: Object
  
  Description: Defines the headers for the table columns, where each key corresponds to the attribute key of the actual data. The key must match the keys in `data` attribute.
   
  Each key takes up to 3 attributes:
  
  - `title`: String
    
    The display name of the header associated to the row key
  - `sortable`: Boolean
  
    If the value of this attribute is sortable.
  - `isDate`: Boolean
      
    If the value of this attribute is a date (for sorting).
-  `data`: Array of Objects
   
   Description: Contains an array of objects, each representing a row of data. The key must match the keys in `headers` attribute.

---
## Table Component `<Table>`

This component takes in 2 properties:
- `tableData`
  
  Description: The table data object described above
- `cssClasses`
  
  Description: Any additional CSS classes to add.
### How to use?
```
<Table tableData={tableData} />
```
